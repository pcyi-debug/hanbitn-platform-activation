# 비개발자를 위한 Git 매뉴얼 — Claude Code 프롬프트 중심

> Git 명령어를 외울 필요 없습니다. **Claude Code에게 한국어로 부탁하면** AI가 대신 실행합니다.
> 이 매뉴얼은 "어떤 상황에서 어떻게 말하면 되는지"를 알려줍니다.

---

## 1. Git이 왜 필요한가요?

Git은 **"작업 내용의 자동 저장 + 타임머신"** 입니다.

- 게임의 **세이브 포인트**처럼, 원하는 시점의 상태를 저장해 둡니다.
- 무언가 잘못돼도 **이전 상태로 되돌릴** 수 있습니다.
- 여러 사람이 **같은 파일을 안전하게** 함께 고칠 수 있습니다.
- **GitHub** = 그 저장본을 인터넷에 올려두는 창고(클라우드). 백업 + 공유 역할.

## 2. 꼭 알아야 할 5가지 개념 (쉽게)

| 용어 | 쉬운 뜻 | 비유 |
|------|---------|------|
| **커밋(commit)** | 지금까지의 변경을 한 묶음으로 저장 | 사진 한 장 찍어 앨범에 넣기 |
| **푸시(push)** | 내 컴퓨터의 저장본을 GitHub에 업로드 | 앨범을 클라우드에 백업 |
| **풀(pull)** | GitHub의 최신본을 내 컴퓨터로 내려받기 | 클라우드 최신 앨범 받기 |
| **브랜치(branch)** | 본 작업과 분리된 별도 작업 공간 | 원본을 두고 사본에서 실험 |
| **저장소(repository)** | 프로젝트 전체와 그 모든 기록 | 프로젝트 전용 앨범 보관함 |

> 가장 자주 쓰는 건 **커밋 → 푸시** 두 가지입니다. "저장하고 → 올린다".

## 3. 상황별 프롬프트 (이 매뉴얼의 핵심)

### 가장 많이 쓰는 것

| 상황 | 이렇게 말하세요 |
|------|-----------------|
| 변경 저장 + 업로드 | `지금까지 작업한 거 커밋하고 푸시해줘` |
| 저장만 (업로드는 나중) | `지금 변경 내용을 커밋해줘` |
| 알아서 메시지까지 | `적절한 메시지로 커밋하고 푸시해줘` |

### 확인하고 싶을 때

| 상황 | 이렇게 말하세요 |
|------|-----------------|
| 뭐가 바뀌었는지 | `지금 어떤 파일이 바뀌었는지 알려줘` |
| 구체적 변경 내용 | `방금 바뀐 내용을 비개발자도 알게 요약해줘` |
| 최근 기록 보기 | `최근 커밋 5개를 한국어로 설명해줘` |
| 업로드됐는지 | `지금 GitHub에 다 올라가 있는 상태야?` |

### 되돌리고 싶을 때

| 상황 | 이렇게 말하세요 |
|------|-----------------|
| 저장 안 한 변경 취소 | `아직 커밋 안 한 변경을 되돌려줘` |
| 방금 커밋 취소 | `방금 한 커밋을 취소하되 작업 내용은 남겨줘` |
| 특정 파일만 원복 | `○○ 파일만 마지막 저장 상태로 되돌려줘` |

> ⚠️ 되돌리기는 작업이 사라질 수 있습니다. AI가 "정말 되돌릴까요?"라고 물으면, 무엇이 사라지는지 확인한 뒤 답하세요.

### 새 작업을 안전하게 시작할 때

| 상황 | 이렇게 말하세요 |
|------|-----------------|
| 별도 공간에서 작업 | `새 브랜치 만들어서 거기서 작업해줘` |
| 본 작업에 합치기 | `이 브랜치 작업을 main에 합쳐줘` |

### 문제가 생겼을 때

| 상황 | 이렇게 말하세요 |
|------|-----------------|
| 상태가 이상함 | `git 상태가 좀 이상해. 확인하고 안전하게 정리해줘` |
| 업로드 실패 | `푸시가 안 돼. 원인 확인하고 해결해줘` |
| 충돌 발생 | `충돌이 났대. 무슨 뜻인지 설명하고 해결 방법을 알려줘` |

## 4. GitHub의 이슈·PR도 프롬프트로

| 상황 | 이렇게 말하세요 |
|------|-----------------|
| 할 일 등록 | `이 작업을 GitHub 이슈로 등록해줘` |
| 이슈 목록 | `지금 열려있는 이슈 목록 보여줘` |
| 변경 제안(PR) | `이 브랜치 내용을 PR(병합 요청)으로 올려줘` |

> 이 프로젝트에서도 개발 작업 15개를 `깃헙 이슈에 등록해줘` 한마디로 만들었습니다.

## 5. 꼭 지킬 안전 수칙

1. **비밀은 올리지 않기** — 비밀번호·API 키·고객 정보는 절대 커밋 금지.
   → 헷갈리면 `이 안에 올리면 안 되는 민감한 정보가 있는지 확인해줘`
2. **푸시 전 확인** — `푸시하기 전에 뭐가 올라가는지 먼저 보여줘`
3. **강제 푸시 주의** — "force push(강제 푸시)"는 남의 작업을 지울 수 있습니다. AI가 권하지 않으면 하지 마세요.
4. **자주 저장** — 큰 작업 하나를 다 끝내고 한 번에 저장하기보다, **의미 있는 단위마다** 커밋하세요.
5. **메시지는 미래의 나를 위해** — `무슨 작업이었는지 알 수 있게 커밋 메시지 써줘`

## 6. 한눈에 보는 흐름

```
작업한다 → "커밋해줘"(저장) → "푸시해줘"(업로드) → GitHub에 안전 보관
                  ↑ 문제 생기면 "되돌려줘"로 복구
```

## 7. 용어 사전 (빠른 참조)

| 용어 | 뜻 |
|------|-----|
| repository (레포/저장소) | 프로젝트와 모든 변경 기록의 보관함 |
| commit (커밋) | 변경을 한 묶음으로 저장한 시점 |
| push (푸시) | 내 저장본을 GitHub에 업로드 |
| pull (풀) | GitHub 최신본을 내려받기 |
| branch (브랜치) | 분리된 작업 공간 |
| merge (머지/병합) | 두 작업 공간을 하나로 합치기 |
| main (메인) | 기본이 되는 본 작업 줄기 |
| origin (오리진) | 연결된 GitHub 저장소의 기본 이름 |
| remote (리모트) | 인터넷에 있는 저장소(보통 GitHub) |
| PR (Pull Request) | "이 변경을 합쳐도 될까요?" 검토 요청 |
