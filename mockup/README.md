# 한빛앤(주) 플랫폼 활성화 대시보드 — 정적 목업

[PRD](../docs/PRD.md) 기반으로 제작한 **빌드 불필요 정적 목업**입니다. (HTML/CSS/JS)
북극성 지표인 **활성화율(첫 결과물 생성 도달률)** 을 끌어올리기 위한 측정·온보딩·리텐션 경험을 화면으로 구현했습니다.

## 실행 방법

별도 빌드/설치가 없습니다. 둘 중 하나로 실행하세요.

1. **바로 열기**: `mockup/index.html` 더블클릭 (브라우저에서 열림)
2. **로컬 서버**(폰트/CDN 안정):
   ```bash
   # 프로젝트 mockup 폴더에서
   python -m http.server 8080
   # 브라우저에서 http://localhost:8080 접속
   ```

> 외부 의존성은 Pretendard 웹폰트(CDN) 하나뿐이며, 오프라인이어도 시스템 폰트로 폴백되어 동작합니다.

```
mockup/

```

## 화면 ↔ PRD 요구사항 매핑

| 화면 (메뉴) | 내용 | PRD 요구사항 |
|------------|------|-------------|
| **활성화 퍼널** | 가입→온보딩→첫 결과물→D1→D7 단계별 전환·누수율, 활성화율 추세, 목표(30→45→55%) 진행 | R1.1, R1.3, R1.4 |
| **코호트 리텐션** | 가입 주차별 D1/D7/D14/D30 리텐션 히트맵, D7 추세(12→17→22%) | R1.2, R1.4 |
| **세그먼트** | 활성 팀 vs 이탈 팀 프로파일 비교 + 인사이트 | R1.4 (팀 단위) |
| **A/B 실험** | 온보딩 실험 목록·표본·전환율·통계적 유의성(p값) | 성공 기준(실험 검증) |
| **온보딩** | 단일 핵심행동 체크리스트·진행률, 빈 상태 템플릿 CTA, 인앱 넛지 | R2.1, R2.2, R2.3 |
| **재참여** | D1/D7 재방문 트리거 시퀀스, 윈백 시나리오 | R3.1, R3.2 |

## 설계 원칙 반영

- **MVP 우선**: 측정·분석 4개 화면(퍼널/코호트/세그먼트/실험)을 사이드바 상단 "측정·분석(MVP)" 그룹으로 분리.
- **팀(워크스페이스) 단위 집계**: 모든 지표를 개인이 아닌 팀 단위로 표기 (`◎ 팀 단위` 태그).
- **북극성 지표 상시 노출**: 상단 KPI 영역에 활성화율 + 선행/후행 지표 4종을 모든 화면 공통 표시.

## 인터랙티브 동작 (이해관계자 데모용)

헤더의 두 필터는 **실제로 화면 수치를 다시 그립니다** — 시연 중 클릭하면 살아있는 대시보드처럼 반응합니다.

- **워크스페이스 셀렉터**: 전체 / 엔터프라이즈(활성화율 52%) / 스타트업(34%) / 신규 가입(22%) 세그먼트별로 NSM·퍼널·팀 수·최대 누수 지점이 바뀝니다. → 세그먼트별 활성화 격차를 한눈에 보여주는 핵심 스토리.
- **기간 셀렉터**: 최근 30일 / 7일 / 이번 분기에 따라 변화량(delta) 배율과 갱신 주기 라벨이 조정됩니다.

> 모든 수치는 PRD/KPI 문서 기반의 **예시(가정) 데이터**이며, 실제 데이터 진단 후 확정됩니다.
