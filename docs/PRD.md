# PRD — 한빛앤(주) 플랫폼 활성화 정체 해결

> 심층 인터뷰(Deep Interview) 기반으로 작성된 제품 요구사항 정의서(Product Requirements Document)
> 최종 모호성 점수: **15.4%** (임계값 20% 통과) · 유형: Brownfield · 인터뷰 6라운드

---

## 1. 개요 (Overview)

한빛앤(주)의 **B2B SaaS 업무툴**은 가입 팀이 **첫 결과물(Deliverable) 생성**이라는 핵심 가치(Aha Moment)에 도달하지 못하고 이탈하면서 활성화 지표가 정체되어 있다. 본 PRD는 **활성화율(가입 팀 중 첫 결과물 생성 도달 비율)** 을 끌어올리기 위한 제품 요구사항을 정의한다.

- **제품 유형**: B2B SaaS / 업무툴 (구독형)
- **핵심 사용자**: 기업 담당자 및 협업 팀
- **Aha Moment**: 가입 후 **첫 결과물 생성**
- **북극성 지표(NSM)**: 활성화율 = 첫 결과물 생성 도달률

## 2. 문제 정의 (Problem Statement)

가입은 유지되나 신규 팀이 **첫 결과물 생성에 도달하기 전에 이탈**하여 활성화 지표가 정체 구간에 머물러 있다. 무엇이 가장 큰 누수 지점인지 **데이터로 규명된 적이 없다**(측정 체계 부재가 근본 원인).

## 3. 목표 (Goals)

| 구분 | 내용 |
|------|------|
| 핵심 목표 | 활성화율(첫 결과물 생성 도달률)을 단계적으로 상승시킨다 |
| 정량 목표 | **30%(baseline) → 45%(Q1) → 55%(Q2)** |
| 선결 과제 | 활성화 퍼널·코호트 **측정 체계**를 먼저 구축해 개선의 성패를 판단 가능하게 한다 |

## 4. 범위 (Scope / Topology)

| 구성요소 | 상태 | 설명 |
|----------|:----:|------|
| **측정·분석 체계** | ✅ Active (MVP) | 활성화 퍼널/코호트 대시보드 + Aha(첫 결과물 생성) 정의. **최우선 단독 출시** |
| **온보딩/활성화 경험** | ✅ Active | 가입~첫 결과물까지 마찰 감소, 단일 핵심행동 유도 |
| **리텐션/재참여** | ✅ Active | 팀 단위 재방문 유도(알림·윈백 등) |
| 성장 루프(추천·개인화) | ⏸ Deferred | 장기 과제. 본 PRD 범위에서 제외(사용자 확정) |

> **실행 순서 원칙**: 자원이 제한적이므로 **측정 체계를 단독으로 먼저 출시**한 뒤, 데이터로 가장 큰 누수 지점을 규명하고 온보딩·리텐션 개입을 순차 적용한다.

## 5. 비범위 (Non-Goals)

- 성장 루프(추천·개인화) 구축 — 차기 PRD로 분리
- 제품 아키텍처/UX 전면 재설계 — **기존 제품 위 증분 개선만** 허용
- 신규 BM·가격 정책 개편, 신규 시장 진출

## 6. 제약 조건 (Constraints)

| 유형 | 제약 |
|------|------|
| 자원/일정 | 소규모 인력, 분기 단위 빠른 실행. 대규모 재구축 불가 |
| 기술 | 기존 제품 위에서만 증분 개선 (아키텍처 대변경 금지) |
| 데이터/보안 | B2B 특성상 고객사 데이터·권한 제약 준수 (알림·실험 포함) |

## 7. 핵심 요구사항 (Requirements)

### 7.1 측정·분석 체계 (MVP, 최우선)
- **R1.1** 활성화 퍼널 대시보드: `가입 → 온보딩 완료 → 첫 결과물 생성 → D1 재방문 → D7 습관화` 단계별 전환율/누수율을 측정한다.
- **R1.2** 코호트 리텐션: 가입 주차별 D1/D7/D30 리텐션을 추적한다.
- **R1.3** Aha 정의 확정: "첫 결과물 생성"의 이벤트 정의를 데이터로 명문화하고 baseline 활성화율을 산출한다.
- **R1.4** 팀 단위 집계: 개인이 아닌 **팀(워크스페이스) 단위**로 활성/이탈을 구분한다.

### 7.2 온보딩/활성화 경험
- **R2.1** 가입 직후 **단일 핵심 행동(첫 결과물 생성)** 으로 유도하는 온보딩 가이드와 진행률 표시.
- **R2.2** 빈 상태(Empty State)에 예시/템플릿/CTA 배치로 Time-to-Value 단축.
- **R2.3** 첫 결과물 미완료 팀 대상 인앱 넛지.

### 7.3 리텐션/재참여
- **R3.1** 첫 결과물 생성 후 재방문(D1/D7) 유도 트리거(알림·이메일) 설계.
- **R3.2** 이탈 팀 대상 윈백 시나리오.

## 8. 성공 기준 (Acceptance Criteria)

- [ ] 활성화 퍼널·코호트 대시보드가 가동되어 단계별 전환율을 일/주 단위로 조회할 수 있다.
- [ ] "첫 결과물 생성" Aha 이벤트가 정의되고 baseline 활성화율이 산출된다.
- [ ] 활성화율이 **Q1 말 45%**, **Q2 말 55%** 목표 추세에 도달한다(정체→상승 반전).
- [ ] 최소 1개 온보딩 개입에 대해 A/B로 첫 결과물 도달률 개선이 통계적으로 검증된다.
- [ ] 모든 개선은 기존 제품 위 증분 변경으로 구현된다(아키텍처 대변경 없음).

## 9. 지표 체계 (Metrics)

| 구분 | 지표 |
|------|------|
| 북극성(NSM) | 활성화율 = 가입 팀 중 첫 결과물 생성 도달률 |
| 선행 지표 | 온보딩 완료율, 첫 결과물 도달률, 넛지 클릭률 |
| 후행 지표 | D7 팀 리텐션, 주간 활성 팀 수 |

상세 정의·대시보드 설계는 [05. KPI / 지표 체계](05-kpi-metrics.md) 참조.
