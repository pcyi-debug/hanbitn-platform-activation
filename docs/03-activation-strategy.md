# 03. 활성화 전략 (Activation Strategy)

원인 분석에서 도출된 우선순위를 바탕으로, **단기·중기·장기** 활성화 전략과 실험 가설을 정의합니다. 모든 전략은 **가설 → 실험 → 측정 → 학습** 사이클로 실행합니다.

## 1. 전략 방향성

> "유입을 늘리기 전에, 들어온 사용자가 가치를 빠르게 경험하고 다시 돌아오게 만든다."

- **Time-to-Value 단축**: 가입 후 가치 경험까지의 시간/단계 최소화
- **Aha Moment 강화**: 핵심 가치 순간을 명확히 정의하고 모든 사용자를 그곳으로 유도
- **습관 루프 구축**: 트리거 → 행동 → 보상 → 투자 사이클로 재방문 정착

## 2. 단기 전략 (0~3개월): 누수 막기

| # | 전략 | 핵심 액션 | 대상 가설 |
|---|------|-----------|-----------|
| S1 | 활성화 측정 체계 구축 | 퍼널·코호트 대시보드 구축, Aha Moment 정의 | 근본 원인 |
| S2 | 온보딩 재설계 | 가입 직후 "단일 핵심 행동" 가이드, 진행률 표시 | B-1, B-2 |
| S3 | 빈 상태 개선 | Empty State에 예시·템플릿·CTA 배치 | B-3 |
| S4 | 첫 행동 넛지 | 첫 핵심행동 미완료자 대상 인앱 안내 | B-1 |

### 단기 실험 가설 예시
- **가설**: 온보딩에서 핵심 행동을 1개로 단순화하면 Aha 도달률이 향상된다
- **실험**: A/B 테스트 (기존 온보딩 vs 단일행동 온보딩)
- **성공 지표**: 첫 핵심행동 도달률 +15%p

## 3. 중기 전략 (3~6개월): 리텐션 구조화

| # | 전략 | 핵심 액션 | 대상 가설 |
|---|------|-----------|-----------|
| M1 | 재참여 트리거 구축 | 행동 기반 이메일/푸시 시퀀스 설계 | D-1 |
| M2 | 세그먼트별 경험 분기 | 사용자 유형별 추천/온보딩 분기 | C-3 |
| M3 | 핵심 기능 발견성 개선 | 기능 추천, 사용 맥락 기반 노출 | C-2 |
| M4 | 윈백 캠페인 | 이탈 사용자 대상 재유입 시나리오 | D-3 |

### 중기 실험 가설 예시
- **가설**: D1 미방문자에게 가치 리마인드 알림을 보내면 D7 리텐션이 상승한다
- **실험**: 알림 발송군 vs 대조군 코호트 비교
- **성공 지표**: D7 리텐션 +5%p

## 4. 장기 전략 (6~12개월): 성장 루프

| # | 전략 | 핵심 액션 |
|---|------|-----------|
| L1 | Growth Loop 설계 | 사용 → 가치 생성 → 공유/추천 → 신규 유입 루프 구축 |
| L2 | 추천(Referral) 구조 | 활성 사용자의 자연 추천 인센티브 설계 |
| L3 | 데이터 기반 개인화 | 행동 데이터 기반 개인화 추천 고도화 |
| L4 | 활성화 자동화 | 라이프사이클 자동 메시징 파이프라인 |

## 5. 전략 ↔ 가설 ↔ 지표 연결표

| 전략 | 검증 가설 | 선행 지표(Leading) | 후행 지표(Lagging) |
|------|-----------|--------------------|--------------------|
| S2 온보딩 재설계 | B-1 | 온보딩 완료율 | 첫 핵심행동 도달률 |
| M1 재참여 트리거 | D-1 | 알림 클릭률 | D7 리텐션 |
| L2 추천 구조 | - | 추천 발송 수 | 추천 경유 가입률 |

## 6. 실행 원칙 (Operating Principles)

1. **측정 없이 실행 없다** — 모든 전략은 측정 가능한 지표와 함께 출시
2. **한 번에 하나의 변수** — 인과 규명을 위해 실험 변수 통제
3. **작게 실험, 빠르게 학습** — MVP 실험 후 확대
4. **정성 + 정량 병행** — 숫자가 "왜"를 말해주지 않으면 인터뷰로 보완
