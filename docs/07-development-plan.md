# 07. 개발 계획문서 (Development Plan)

> [PRD](PRD.md)를 **실제 제품 MVP**로 구현하기 위한 개발자용 실행 계획.
> 각 작업(Task)은 GitHub 이슈로 등록되며, 이슈에는 **작업 배경 · 작업 내용 · 인수 조건 · 의존성**이 포함된다.
> UI 레퍼런스: [`mockup/`](../mockup/) (정적 목업)

---

## 1. 개발 원칙 (Engineering Principles)

- **측정 우선(Measurement-first)**: PRD 근본 원인은 "측정 체계 부재". 대시보드/개입보다 **이벤트 스키마·수집 파이프라인을 먼저** 구축한다. 측정이 없으면 어떤 실험도 성패를 판단할 수 없다.
- **팀(워크스페이스) 단위**: 모든 활성/이탈/리텐션은 개인이 아닌 **팀 단위**로 집계한다.
- **증분 개선만**: 기존 제품 위 증분 변경. 아키텍처 대변경 금지(PRD 제약).
- **Build vs Buy 검토**: 퍼널/코호트/세그먼트/실험은 제품분석 툴(PostHog/Amplitude 등) 기본 기능. 직접 구축 전 도입을 먼저 평가한다.
- **데이터/보안 준수**: B2B 특성상 고객사 데이터 격리·권한·동의를 모든 수집/알림/실험에 적용한다.
