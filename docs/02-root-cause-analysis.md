# 02. 원인 분석 (Root Cause Analysis)

활성화 정체는 단일 원인보다 **유입 → 온보딩 → 가치경험 → 재참여** 사슬의 여러 약한 고리가 겹쳐 발생합니다. 본 문서는 세 가지 관점(데이터·고객·구조)에서 가설을 정리합니다.

## 1. 분석 프레임: 활성화 퍼널 분해

```
가입 완료 ──▶ 온보딩 완료 ──▶ 첫 핵심행동(Aha) ──▶ 재방문(D1) ──▶ 습관화(D7+)
   100        ?              ?                  ?            ?
```

각 단계 전환율을 측정하여 **가장 큰 누수 지점**을 찾는 것이 분석의 출발점입니다.

## 2. 가설 트리 (Hypothesis Tree)

### 가설 A. 유입 품질 문제 (Acquisition Quality)
- A-1. 타겟과 맞지 않는 채널에서 유입되어 의도 없는 사용자가 많다
- A-2. 광고 메시지와 실제 제품 가치의 **기대 불일치(Mismatch)** 가 크다
- A-3. 특정 캠페인 유입 코호트의 리텐션이 구조적으로 낮다

### 가설 B. 온보딩·첫 경험 문제 (Onboarding)
- B-1. 가입 후 **무엇을 해야 할지 모호** 하여 첫 행동까지 마찰이 크다
- B-2. 가치를 느끼기까지 단계가 너무 많다 (Time-to-Value 과다)
- B-3. 빈 상태(Empty State)에서 동기 부여 요소가 부족하다

### 가설 C. 핵심 가치 전달 문제 (Value Delivery)
- C-1. **Aha Moment 자체가 불명확** 하거나 사용자에게 약하게 전달된다
- C-2. 핵심 기능 발견성(Discoverability)이 낮다
- C-3. 사용자 세그먼트별로 필요한 가치가 다른데 단일 경험만 제공한다

### 가설 D. 재참여 구조 문제 (Re-engagement)
- D-1. 다시 돌아올 **트리거(알림·이메일·푸시)** 가 약하거나 없다
- D-2. 재방문해도 새로운 가치가 없어 습관이 형성되지 않는다
- D-3. 이탈 사용자 대상 윈백(Win-back) 시나리오 부재

## 3. 원인 우선순위 매트릭스

각 가설을 **영향도 × 검증 용이성** 으로 평가하여 우선 검증 대상을 선정합니다.

| 가설 | 영향도 | 검증 용이성 | 우선순위 |
|------|:------:|:-----------:|:--------:|
| B-1 온보딩 모호성 | 높음 | 높음 | **1순위** |
| C-1 Aha Moment 불명확 | 높음 | 중간 | **2순위** |
| D-1 재참여 트리거 부재 | 높음 | 높음 | **3순위** |
| A-2 기대 불일치 | 중간 | 중간 | 4순위 |
| C-2 기능 발견성 | 중간 | 높음 | 5순위 |

> 우선순위는 실제 데이터 진단 후 재조정합니다.

## 4. 검증 방법 (Validation)

| 방법 | 검증 대상 | 산출물 |
|------|-----------|--------|
| 퍼널 분석 | 단계별 전환율/누수 | 퍼널 리포트 |
| 코호트 분석 | 가입 시점별 리텐션 곡선 | 리텐션 히트맵 |
| 세그먼트 분석 | 활성 vs 이탈 사용자 특성 | 세그먼트 프로파일 |
| 사용자 인터뷰 | 이탈 사유 정성 파악 | 인터뷰 노트 (5~7명) |
| 세션 리플레이 | 온보딩 마찰 지점 | 마찰 지점 목록 |

## 5. 5 Whys 예시 (온보딩 누수)

1. **왜** 가입 후 첫 핵심행동 도달률이 낮은가? → 사용자가 무엇을 해야 할지 모름
2. **왜** 무엇을 해야 할지 모르는가? → 온보딩이 기능 설명 위주이고 행동 유도가 없음
3. **왜** 행동 유도가 없는가? → "첫 가치 경험"을 명확한 단일 행동으로 정의하지 않음
4. **왜** 정의하지 않았는가? → Aha Moment를 데이터로 규명한 적이 없음
5. **왜** 규명하지 않았는가? → 활성화 측정 체계가 부재 → **근본 원인**

> 결론: **활성화 측정 체계 부재**가 여러 증상의 공통 뿌리일 가능성이 높음 → [05. KPI 체계](05-kpi-metrics.md) 우선 구축 필요.
