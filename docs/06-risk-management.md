# 06. 리스크 관리 (Risk Management)

활성화 개선 프로젝트 실행 시 발생 가능한 리스크를 식별하고 대응 계획을 정의합니다.

## 1. 리스크 등록부 (Risk Register)

| ID | 리스크 | 발생 가능성 | 영향도 | 대응 전략 |
|----|--------|:-----------:|:------:|-----------|
| R1 | 측정 체계 미비로 실험 성패 판단 불가 | 높음 | 높음 | Q1 최우선 구축, 데이터 검증 선행 |
| R2 | 표본 부족으로 실험 통계적 유의성 미달 | 중간 | 높음 | 사전 표본 크기 산정, 실험 기간 확보 |
| R3 | 단기 지표 개선이 장기 리텐션과 무관 | 중간 | 중간 | 선행·후행 지표 동시 추적 |
| R4 | 잦은 변경으로 사용자 경험 혼란 | 중간 | 중간 | 변경 통제, 점진적 롤아웃 |
| R5 | 부서 간 우선순위 충돌 | 중간 | 중간 | 북극성 지표로 정렬, 주간 리뷰 |
| R6 | 알림/메시징 과다로 이탈 가속 | 낮음 | 높음 | 빈도 상한·옵트아웃 보장 |

## 2. 리스크 대응 원칙

- **선행 지표 모니터링**: 후행 지표가 나빠지기 전에 조기 경고
- **점진적 롤아웃**: 전체 배포 전 일부 사용자에 우선 적용
- **롤백 기준 사전 정의**: 핵심 지표 악화 시 즉시 되돌리는 임계치 설정

## 3. 안티패턴 (피해야 할 것)

1. **유입만 늘리기** — 새는 독에 물 붓기. 활성화 개선이 먼저
2. **허무 지표 자축** — 누적 가입자 증가에 안주
3. **한 번에 여러 변수 변경** — 인과 규명 불가
4. **정량만 보기** — "왜"를 모른 채 숫자만 좇기
5. **단발 캠페인 의존** — 구조 개선 없이 일시적 부양

## 4. 의사결정 체크포인트

| 시점 | 점검 질문 |
|------|-----------|
| 실험 시작 전 | 성공/실패 기준과 표본 크기가 정의됐는가? |
| 실험 중 | 선행 지표가 예상대로 움직이는가? |
| 실험 종료 | 통계적으로 유의하며 후행 지표로 이어지는가? |
| 확대 적용 전 | 부작용(다른 지표 악화)은 없는가? |
