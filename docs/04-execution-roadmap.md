# 04. 실행 로드맵 (Execution Roadmap)

전략을 분기별 실행 단위로 분해하고, 책임(R&R)과 산출물을 명시합니다.

## 1. 분기별 로드맵

### Q1 — 진단 및 기반 구축
| 주차 | 작업 | 산출물 |
|------|------|--------|
| W1–2 | 활성화 퍼널/코호트 대시보드 구축 | 데이터 대시보드 |
| W3–4 | Aha Moment 데이터 규명 | Aha 정의 문서 |
| W5–8 | 온보딩 재설계 (S2) A/B 실험 | 실험 리포트 |
| W9–12 | 빈 상태·첫 행동 넛지 (S3, S4) | 개선 배포 |

### Q2 — 리텐션 구조화
| 주차 | 작업 | 산출물 |
|------|------|--------|
| W1–4 | 재참여 트리거 시퀀스 (M1) | 메시징 파이프라인 |
| W5–8 | 세그먼트별 경험 분기 (M2) | 세그먼트 분기 로직 |
| W9–12 | 윈백 캠페인 (M4) | 윈백 시나리오 |

### Q3–Q4 — 성장 루프
| 주차 | 작업 | 산출물 |
|------|------|--------|
| Q3 | Growth Loop·추천 구조 설계 (L1, L2) | 루프 설계 문서 |
| Q4 | 개인화·자동화 고도화 (L3, L4) | 자동화 파이프라인 |

## 2. 마일스톤

- 🏁 **M1 (Q1 종료)**: 활성화 측정 체계 가동 + 온보딩 실험 1차 결과
- 🏁 **M2 (Q2 종료)**: D7 리텐션 추세 반전 확인
- 🏁 **M3 (Q4 종료)**: 자생적 성장 루프 1개 이상 가동

## 3. 책임 분장 (R&R)

| 영역 | 주관 | 협업 |
|------|------|------|
| 데이터 측정/분석 | 데이터 분석 | 개발 |
| 온보딩/UX | 프로덕트/디자인 | 개발 |
| 재참여 메시징 | 그로스/마케팅 | 데이터 |
| 실험 설계/검증 | 프로덕트 | 데이터 |

## 4. 의존성 및 선후 관계

```
[측정 체계 구축] ──┬─▶ [온보딩 실험] ──▶ [세그먼트 분기]
                   └─▶ [재참여 트리거] ──▶ [윈백 캠페인]
                                          └─▶ [성장 루프]
```

> **핵심 의존성**: 측정 체계가 없으면 모든 실험의 성패를 판단할 수 없으므로 **Q1 W1–2 최우선**.

## 5. 실행 거버넌스

- **주간 그로스 리뷰**: 핵심 지표 추세 + 진행 중 실험 점검
- **격주 실험 회고**: 가설 검증 결과 → 다음 실험 우선순위 갱신
- **분기 전략 리뷰**: 로드맵 재조정 및 자원 재배분
